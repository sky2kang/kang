# 영웅문 [0150] 조건검색식 입력 가이드 (10선 · 클릭 순서)

> 이 문서는 [조건검색식_추천10.md](조건검색식_추천10.md)의 10개 검색식을 **영웅문4 HTS에서
> 실제로 입력하는 클릭 순서**를 단계별로 정리한 것입니다. 키움 OpenAPI로는 조건식을 만들 수
> 없으므로 반드시 영웅문에서 직접 만들어 저장해야 합니다.
>
> ⚠️ 지표 이름·다이얼로그 문구는 영웅문 버전에 따라 조금씩 다를 수 있습니다. 가장 확실한
> 방법은 **지표 검색창에 지표명을 직접 입력**해 찾는 것입니다(0번 공통 참조).

---

## 0. 공통 — 화면 구성과 기본 조작 (처음 1회 숙지)

### 0-1. 조건검색 화면 열기
1. 영웅문 로그인 → 상단 입력창에 **`0150`** 입력 후 Enter → **[0150] 조건검색** 창이 열림.
2. 좌측 상단 **[새로작성]** 버튼 클릭 → 빈 조건식 시작.

### 0-2. 화면 4영역
- **① 좌측 지표 트리**: 시세분석 / 기술적분석 / 재무분석 / 순위분석 … 대분류 → 중분류 → 지표.
- **② 지표 검색창**(트리 상단): 지표명을 입력하면 트리에서 바로 찾아줌. (예: `골든크로스`, `거래대금`)
- **③ 지표 설정 팝업**: 지표를 더블클릭하면 값 입력 팝업이 뜸 → 값 설정 후 **[추가]**.
- **④ 하단 조건식 목록(A, B, C…)**: 추가된 조건이 행으로 쌓임. 행을 더블클릭하면 **수정**, 우측 **[삭제]**.

### 0-3. 지표 1개 추가하는 기본 흐름
> 좌측 트리(또는 검색창)에서 **지표 더블클릭** → 팝업에서 **값 입력** → **[추가]** 클릭 → 하단에 `A`, `B`, `C`… 로 쌓임.

### 0-4. 조건 결합 (AND / OR / 괄호)
- 하단 조건식 목록 위의 **수식 입력란**에 `A and B and C` 형태로 표시됨.
- 기본은 **and(그리고)**. 모든 조건을 동시에 만족시키려면 and 유지.
- `or`나 괄호 `(A and B) or C` 도 직접 편집 가능.

### 0-5. 대상 종목 / 제외 설정 (모든 검색식 공통 권장)
1. 우측 상단 **[대상변경]** 클릭.
2. **포함**: 거래소(코스피) + 코스닥 체크.
3. **제외**: 관리종목 / 투자경고·위험 / 거래정지 / 우선주 / ETF / ETN / 스팩(SPAC) / 정리매매 체크.
4. (선택) **시가총액** 500억 이상 등 → 별도 지표 `시세분석 > 시가총액`으로도 추가 가능.

### 0-6. 저장 & 자동매매 연동
1. 하단 **[조건식 저장]** → 이름 입력(예: `골든크로스_5_20`) → 저장.
2. 본 프로그램 **조건식 시뮬레이션** 탭 → [조건식 불러오기] → 검색식 선택 → 과거 백테스트.
3. **조건 자동매매** 탭 → 사용 체크 + 룰(손절/익절/트레일링/분할) 입력 → 시작.

## 1. 급등주 포착 (거래대금 + 등락률) — 실시간 전용

| 순서 | 지표 트리 위치 (또는 검색창 입력) | 팝업 설정값 | 동작 |
|---|---|---|---|
| A | 시세분석 > 거래대금 > **거래대금** (검색: `거래대금`) | `0`봉전 거래대금 `50000` 백만(=500억? 주의: 단위 백만원) → 5,000(=50억)** 이상 | [추가] |
| B | 시세분석 > 순위·등락 > **전일대비등락률** (검색: `등락률`) | `0`봉전 전일대비등락률 `3`% 이상 `15`% 이하 | [추가] |
| C | 시세분석 > 거래량 > **전일동시간대대비 거래량비율** (검색: `거래량비율`) | 전일동시간대 대비 `200`% 이상 | [추가] |
| D | 시세분석 > 가격 > **현재가** (검색: `현재가`) | 현재가 `1000`원 이상 | [추가] |

> ** 거래대금 단위 주의: 영웅문은 보통 **백만원** 단위. 50억 = `5000`(백만원). 화면 단위 표기를 꼭 확인하세요.

- 수식: `A and B and C and D`
- 저장명 예: `급등주_거래대금50억`
- 자동매매 룰: 손절 -2.5 · 익절 4 · 트레일링 2 · 분할 1

## 입력 시 자주 막히는 포인트

| 증상 | 원인 / 해결 |
|---|---|
| 지표를 트리에서 못 찾음 | 트리 상단 **지표 검색창**에 한글 지표명(예 `골든크로스`, `거래대금`) 입력 |
| 거래대금이 너무 작/큼 | 단위는 보통 **백만원**. 50억 = `5000`, 30억 = `3000` |
| "N봉전" 의미 | `0`봉전 = 오늘(현재), `1`봉전 = 어제. 크로스 판정은 0~1봉 비교 |
| 골든크로스가 두 줄로 표현 | "주가이동평균비교" 팝업의 연산 드롭다운에서 **골든크로스** 선택 (한 줄로 처리) |
| 단발 조회/시뮬레이션 실패 | 상한가·당일거래량·체결강도 기반(1·2·10번)은 **실시간 전용**. 일봉 기반(3·4·5·6·7·8·9번)만 단발·백테스트 가능 |
| 결과가 너무 많음/적음 | 거래대금·거래량비율·등락률 임계값을 올리거나(적게) 내려(많게) 조정 |

> 처음에는 **3·6·9번**(골든크로스·세력매집·RSI반등)처럼 단발 조회·백테스트가 되는 검색식으로
> 시작해 **시뮬레이션 탭에서 과거 성과를 확인**한 뒤 자동매매에 투입하는 것을 권장합니다.
